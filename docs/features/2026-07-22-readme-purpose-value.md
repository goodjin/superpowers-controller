# README 开篇：目的与价值主张

日期：2026-07-22

## 目标

优化 README 开篇，让读者先看清产品定位、快速安装方式，以及为什么要做这个插件。

## 改动范围

- 文件：`README.md`
- 只调整开篇结构与文案；安装细节、设计理念、组成部分、配置等后续章节保留。

## 开篇结构

1. 产品定位（一行说清是什么）
2. 快速安装（一行命令 + 启动方式）
3. 价值主张（四条目的）

## 价值主张文案要点

1. 一次安装并配置好 Superpowers 框架；Agent 主动提醒模型使用相应技能，无需手动触发。
2. 插件自动创建子会话执行各技能；子会话与主控会话隔离，避免污染上下文。
3. 任务相关文档与工作流持久化；系统重启后可从断点恢复，不丢任务。
4. 按任务类型提醒采用不同工作流，也允许模型自定义工作流。

## 验收

- README 开头先见定位、安装、价值主张，再进入详细使用与设计说明。
- 四条价值主张表达清楚，不与后文「设计理念」明显打架。
